## 1. 产品概述

### 1.1 背景与动机

Aiops 是一个 AI 内容运营平台，已具备文案生成（DeepSeek AI）、视频制作（LibTV/Wan）、多平台发布等核心能力。当前用户在创作中文文案后，若需制作多语言音频内容（如播客、短视频配音、海外社交媒体推广），必须在外部工具（如 Google TTS、剪映配音等）中手动完成翻译和语音合成，流程割裂、效率低下。

**痛点**：
- 中文文案 → 翻译 → 语音合成 需跨多个平台完成，来回切换成本高；
- 外部 TTS 工具要么收费高昂（如 AWS Polly、Azure TTS），要么语音质量差（开源方案）；
- 翻译和 TTS 结果无法沉淀在 Aiops 平台内，后续复用时仍需重复操作。

### 1.2 产品目标

在 Aiops 中新增「文案转语音」（TTS）功能，支持：
1. **AI 翻译**：调用 DeepSeek API 将中文文案高质量翻译为 7 种目标语言（中文直读免翻译）；
2. **高质量语音合成**：通过 Microsoft Edge TTS Neural 引擎合成自然流畅的语音；
3. **音频下载**：以 MP3 格式提供下载，方便用于短视频配音、播客、广告等场景；
4. **历史复用**：支持从已有的文案历史中一键选取，快速转语音。

### 1.3 用户角色

| 角色 | 描述 |
|------|------|
| 内容运营 | 需要将文案制作为多语言音频，用于短视频、播客、社交平台推广 |
| 市场营销 | 需要快速产出多语言广告配音素材 |
| 产品经理 | 需要生成产品介绍音频，辅助演示和宣传 |

### 1.4 核心指标（North Star Metrics）

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| TTS 月生成次数 | ≥ 200 次/月 | 后端 API 调用计数 |
| 翻译+TTS 成功率 | ≥ 95% | 成功/总请求 |
| 平均生成耗时 | ≤ 15 秒（短文案） | API 响应时间 P95 |
| 用户激活率 | ≥ 30% 月活用户使用过 TTS | 使用过 TTS 的去重用户数 / 月活用户数 |

## 2. 用户故事

### US-01：一键生成中文语音（P0）

> **作为**内容运营，**我想要**输入中文文案后直接生成中文语音，**以便**快速制作配音素材，无需翻译。

**场景**：
- 用户已有中文文案（"欢迎关注我们的频道，今天带来 AI 最新资讯"）
- 选择目标语言「中文」、语音角色「晓晓（女）」
- 点击「生成语音」
- 立即听到合成的中文语音，并可下载 MP3

**验收要点**：
- 中文目标语言时，跳过翻译步骤，直接 TTS 合成
- 合成结果可播放、可下载

> **作为**市场营销，**我想要**将中文文案翻译成英文并生成英文语音，**以便**制作海外社交媒体推广内容。

**验收要点**：
- 翻译自然流畅，适合口语朗读（非书面翻译腔）

- 音频可播放、下载

## 3. 功能模块拆分

### 3.1 系统架构概览

```
┌──────────────────────────────────────────────────────────────┐
│                       前端 (React + TypeScript)                │
│  TtsPage.tsx — 文案输入 / 语言选择 / 语音选择 / 语速调节 /     │
│                 音频播放 / 下载 / 历史文案面板                  │
└──────────────────────────┬───────────────────────────────────┘
                           │ POST /api/tts/generate
                           │ GET  /api/tts/voices
                           │ GET  /api/tts/download/:id
┌──────────────────────────┴───────────────────────────────────┐
│                    后端 (Express, Node.js)                     │
│  tts.cjs — 路由模块                                          │
│  ├── 参数校验（text 必填）                                    │
│  ├── 翻译 (DeepSeek Chat API)                                │
│  ├── TTS 合成 (edge-tts CLI, Microsoft Edge TTS Neural)      │
│  ├── 文件存储 (data/tts_{id}.mp3)                            │
│  └── 记录持久化 (SQLite, contents 表)                         │
└──────────────────────────┬───────────────────────────────────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
         DeepSeek API  edge-tts CLI  SQLite
         (翻译引擎)    (TTS 引擎)   (持久化)
```

### 3.2 后端模块

| 模块 | 文件 | 职责 |
|------|------|------|
| **TTS 路由** | `server/routes/tts.cjs` | 定义 3 个 API 端点，编排翻译→TTS→存储流程 |
| **语音列表** | `GET /api/tts/voices` | 返回 8 种语言 × 20+ 语音角色的结构化列表 |
| **生成端点** | `POST /api/tts/generate` | 接受 `{text, targetLang, voice, speed}`，执行翻译 + TTS，返回 MP3 URL |
| **下载端点** | `GET /api/tts/download/:id` | 流式返回 MP3 文件，含 Content-Disposition 头 |
| **翻译服务** | 内联在 tts.cjs 中 | 调用 DeepSeek `/v1/chat/completions`，系统提示词优化为口语化翻译 |
| **TTS 引擎** | `edge-tts` CLI | 调用 Microsoft Edge TTS Neural，通过 stdin/text 参数传入文本，输出 MP3 |
| **认证** | `authMiddleware` | 所有端点均需 JWT 认证 |
| **数据存储** | `db.cjs` (SQLite) | 生成记录写入 `contents` 表，type='tts'，含原文、译文、语言、语音、文件路径 |

### 3.3 前端模块

| 模块 | 文件 | 职责 |
|------|------|------|
| **TTS 主页面** | `panel/src/pages/TtsPage.tsx` | 完整 UI：输入区、语言选择器、语音角色选择器、语速滑块、生成按钮、结果展示（音频播放器 + 原文/译文对照 + 下载） |
| **路由注册** | `panel/src/App.tsx` | `/tts` 路由，含 ErrorBoundary |
| **侧边栏入口** | `panel/src/components/Layout.tsx` | 导航菜单添加「语音合成」项（Mic 图标） |
| **历史文案面板** | TtsPage.tsx 右侧 | 展示 contentStore 中最近 20 条有文本内容的记录，支持一键填入 |

### 3.4 数据流

```
用户点击「生成语音」
  → 前端校验 text 非空
  → POST /api/tts/generate { text, targetLang, voice, speed }
  → 后端 authMiddleware 校验 JWT
  → 参数校验（text 必填）
  → [若 targetLang ≠ zh-CN] DeepSeek 翻译
  → edge-tts 合成 MP3 → 保存到 data/tts_{uuid}.mp3
  → 写入 contents 表（type='tts'）
  → 返回 JSON { id, original, translated, lang, voice, url }
  → 前端展示音频播放器 + 原文/译文 + 下载按钮
```

## 4. 技术方案摘要

> **注意**：本节为技术概要，供开发参考，具体实现代码不在此 PRD 范围内。

### 4.1 后端

| 组件 | 方案 | 说明 |
|------|------|------|
| 框架 | Express (Node.js) | 已有，复用 |
| 翻译 | DeepSeek Chat API (`deepseek-chat` 模型) | 已有 API Key 配置，temperature=0.3 确保翻译稳定性 |
| TTS | Microsoft Edge TTS (`edge-tts` CLI) | 免费、神经网络语音、多语言、高质量 |
| 存储 | 文件系统 `data/tts_{uuid}.mp3` + SQLite `contents` 表 | 已有，复用 |
| 认证 | JWT (`authMiddleware`) | 已有，复用 |
| 新增依赖 | `edge-tts` (Python CLI) | 需在服务器上预先安装 |

**关键设计决策**：

1. **中文直读跳过翻译**：当 `targetLang === 'zh-CN'` 时，不调用 DeepSeek，原文直接传入 TTS，节省 API 成本和时间。

2. **文本清洗**：在传入 edge-tts 前对特殊字符做规范化处理（弯引号→直引号、换行→空格），避免 CLI 解析异常。

3. **任务记录**：每次生成写入 `contents` 表（`type='tts'`），包含原文、译文、语言、语音、文件路径，便于后续审计和复用。

4. **并发安全**：每个请求生成唯一的 `uuid` 作为文件名，避免并发写入同一文件。

### 4.2 前端

| 组件 | 方案 | 说明 |
|------|------|------|
| 框架 | React 18 + TypeScript | 已有 |
| 路由 | React Router v6 | 已有，新增 `/tts` 路由 |
| 状态管理 | Zustand (`contentStore`) | 已有，复用历史文案数据 |
| 样式 | TailwindCSS | 已有，保持一致设计语言 |
| 音频播放 | HTML5 `<Audio>` API | 浏览器原生支持，无需额外依赖 |

## 7. 附录

### 7.1 术语表

| 术语 | 说明 |
|------|------|
| TTS | Text-to-Speech，文本转语音 |
| Edge TTS | Microsoft Edge 浏览器的神经网络语音合成引擎，通过 `edge-tts` CLI 调用 |
| Neural Voice | 基于深度神经网络的语音合成，音质自然流畅 |
| 语速 (Rate) | 语音播放速度，-50% 为最慢，+100% 为最快 |
| MP3 | MPEG Audio Layer III，通用音频压缩格式 |

### 7.2 已知限制

1. **edge-tts 依赖 Internet 连接**：需要访问 Microsoft Edge TTS 服务，离线环境不可用
2. **翻译质量依赖 DeepSeek**：翻译质量取决于 DeepSeek Chat 模型能力
3. **单次生成限制**：不支持超长文案（>5000 字符），建议用户分段处理
4. **无流式响应**：当前为同步等待完整 MP3 文件生成后返回，用户需等待全过程
