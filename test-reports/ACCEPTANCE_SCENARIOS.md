# Aiops SAAS 验收场景清单
> 基于 project-config.md 的 PRD → E2E 验收场景
> 版本: v1.0 | 日期: 2026-06-29
> 测试服务器: 43.156.78.59:5290

## 验收范围
全部 8 个前端页面 + 核心 API 流程

---

### AS-001: Landing 首页加载
**类型**: 页面加载 + 首屏
**操作步骤**:
1. browser open http://43.156.78.59:5290/
2. snapshot 首屏（Hero 区域）
3. 滚动到 Features 区域 → snapshot
4. 滚动到 Pricing 区域 → snapshot
5. 滚动到 Stats 区域 → snapshot
6. 滚动到 Footer → snapshot
**期望**: 页面 200，首屏有标题/描述/CTA 按钮，各区块渲染完整

### AS-002: Landing 导航栏
**类型**: 交互 + 导航
**操作步骤**:
1. 在首页 → snapshot Navbar
2. 点击 "登录" 按钮 → 应跳转到 /login
3. snapshot 登录页
4. browser navigate 回到 /
5. 点击 "注册" 或 "开始使用" CTA → 应跳转到 /register
6. snapshot 注册页
**期望**: 导航跳转正确，URL 变化，页面渲染无误

### AS-003: 注册流程
**类型**: 表单交互 + API
**操作步骤**:
1. browser open http://43.156.78.59:5290/register
2. snapshot 注册表单
3. 不填任何字段直接点注册 → 应显示验证错误
4. 填写无效邮箱（如 "abc"）→ 应显示邮箱格式错误
5. 填写短密码（<6 位）→ 应提示密码要求
6. 填写符合要求的注册信息 → 点注册
7. snapshot 结果（成功提示或跳转）
**期望**: 表单验证到位，成功注册后跳转登录页

### AS-004: 登录流程
**类型**: 表单交互 + API + Token
**操作步骤**:
1. browser open http://43.156.78.59:5290/login
2. snapshot 登录表单
3. 空表单提交 → 应显示验证错误
4. 错误密码 → 应显示"用户名或密码错误"
5. 正确凭据登录 → snapshot 成功跳转
6. 确认跳转到 /dashboard
7. snapshot 仪表盘页面
**期望**: 登录成功获取 JWT，跳转仪表盘，页面有用户信息

### AS-005: Dashboard 仪表盘
**类型**: 页面渲染 + 数据加载
**操作步骤**:
1. 登录后到达 /dashboard
2. snapshot 仪表盘全貌
3. 查看概览数据（API 用量/配额等）
4. 点击左侧/顶部导航各项
5. snapshot 每个 Tab 切换后的状态
**期望**: 仪表盘数据加载正常，导航切换流畅

### AS-006: 内容管线 Pipeline
**类型**: 核心功能全链路
**操作步骤**:
1. 登录后访问 /pipeline
2. snapshot 管线页面
3. 选择内容生成 Tab → 输入文案需求 → 点生成
4. snapshot 生成结果
5. 对结果进行操作（优化/翻译/TTS 合成）
6. 切换到不同的管线阶段
**期望**: 管线各步骤可操作，生成结果可复制/下载

### AS-007: 配额消费验证
**类型**: 功能 + API 验证
**操作步骤**:
1. 登录 → 记录当前配额（API 查询 /api/quota/status）
2. 执行一次文案生成（/api/content/generate）
3. 再次查询配额 → 确认配额减少
4. 配额不足时的错误提示
**期望**: 配额正确扣减，余额不足时给出明确提示

### AS-008: Token 过期处理
**类型**: 边界 + 错误处理
**操作步骤**:
1. 正常登录
2. 手动用过期 Token 请求 /api/auth/me → 确认 401
3. 前端在 Token 过期后是否自动跳转登录页
4. 重新登录后恢复正常
**期望**: Token 过期返回 401，前端弹出登录或自动跳转

### AS-009: 设置页 Settings
**类型**: 页面 + 表单
**操作步骤**:
1. 登录后访问 /settings
2. snapshot 设置页
3. 切换各 Tab（安全/账单/API Keys）
4. 修改密码功能 → 输入旧密码/新密码/确认密码
5. 密码修改验证（空值/不一致/旧密码错误）
6. API Keys 页面查看/创建/删除 Key
**期望**: 设置项可正常操作，密码修改有验证和反馈

### AS-010: 响应式布局（移动端）
**类型**: 响应式
**操作步骤**:
1. browser resize 375px 宽度
2. 访问首页 → snapshot 移动版
3. 访问登录页 → snapshot 移动版
4. 访问仪表盘 → snapshot 移动版
5. 导航菜单是否折叠为汉堡菜单
**期望**: 375px 下布局正常，无横向滚动，功能可操作

### AS-011: 404 页面
**类型**: 边界
**操作步骤**:
1. browser open http://43.156.78.59:5290/nonexistent-page
2. snapshot 404 页面
3. 确认有"回到首页"按钮
4. 点击按钮 → 回到 /
**期望**: 404 页面友好，有返回链接

### AS-012: 错误状态覆盖
**类型**: 边界 + 体验
**操作步骤**:
1. 未登录直接访问 /dashboard → 应跳转登录页
2. 未登录访问 /pipeline → 应跳转登录页
3. 未登录访问 /settings → 应跳转登录页
4. API 返回错误时页面是否白屏（检查各 loading/error 状态）
**期望**: 未登录受保护路由跳转登录，API 错误不白屏
