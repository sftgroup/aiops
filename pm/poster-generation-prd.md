## 1. 功能背景与目标

### 1.1 背景
Aiops 文案运营平台已具备 TTS 语音合成和视频生成（Seedance + ffmpeg logo 水印）能力。运营团队在日常工作中频繁需要为社交媒体、活动推广、产品宣传生成海报，目前依赖设计师手动制作，效率低且周期长。

### 1.2 目标
- 用户输入主题或文案，AI 自动生成高质量海报图片
- 生成海报自动叠加品牌 Logo 水印
- 海报可直接用于社交媒体（小红书、微信、抖音等）发布
- 将海报生成能力整合到现有 Aiops 平台，形成「文案 → 语音 → 视频 → 海报」完整内容生产链路

### 1.3 目标用户
- 内容运营、活动运营、品牌推广人员
- 需要高频产出视觉素材但无设计技能的用户

## 2. 用户故事

| ID | 角色 | 需求 | 目的 |
|----|------|------|------|
| US-1 | 内容运营 | 输入一段文案，快速生成带品牌 Logo 的配图海报 | 省去找设计师排期的时间 |
| US-2 | 活动运营 | 输入活动主题（如"618大促"），生成多张不同风格的海报备选 | 快速验证不同视觉方向 |
| US-3 | 品牌运营 | 生成海报自动叠加品牌 Logo 水印，支持指定 Logo 位置和大小 | 保证品牌视觉规范 |
| US-4 | 运营主管 | 查看历史生成记录，复用或调整已生成的海报 | 减少重复工作 |

## 3. AI 服务选型建议

### 3.1 备选方案

| 方案 | API | 优势 | 劣势 | 推荐度 |
|------|-----|------|------|--------|
| A | 火山引擎 Seedance 图像生成 | 与现有视频生成同供应商，统一计费 | MLA 可能较基础 | ⭐⭐⭐⭐ |
| B | 火山引擎即梦（Jimeng）API | 专注图像生成，中文语义理解好 | 新接入需评估 | ⭐⭐⭐⭐⭐ |
| C | Stable Diffusion（自建） | 完全可控，可微调 | 运维成本高 | ⭐⭐⭐ |
| D | Midjourney / DALL·E | 生成质量高 | 中文支持弱，需翻墙 | ⭐⭐ |

### 3.2 推荐方案

**已实施：火山引擎即梦 API（方案 B）** ✅

- 模型：`doubao-seedream-3.0-lite-251215`（DEFAULT），备选 `doubao-seedream-3.0-t2i-251215`
- 统一通过 Volcengine Ark API 调用（`ark.cn-beijing.volces.com/api/v3/contents/generations/tasks`）
- 与 Seedance 视频共用 Ark API Key（`SEEDANCE_API_KEY` / `ARK_API_KEY`），无需新增 Key
- Logo 水印复用现有 ffmpeg overlay 逻辑（PNG 叠加到图片，通过单帧处理）
- 实现文件：`server/poster-api.cjs`

## 4. 详细功能规格

### 4.1 输入

| 参数 | 类型 | 必填 | 说明 | 实现状态 |
|------|------|------|------|---------|
| 主题/文案 | string | 是 | 海报核心文案，200 字以内 | ✅ |
| 海报风格 | enum | 否 | default/tech/minimal/festive/promo/handdrawn | ✅ |
| 海报尺寸 | enum | 否 | square/portrait/story/landscape/wide/gzh，默认 portrait | ✅ |
| Logo | file | 否 | 使用平台预设品牌 Logo（video_logo_url） | ✅ |
| Logo 位置 | enum | 否 | 固定右上角 overlay=W-w-20:20 | ✅ |
| Logo 大小 | enum | 否 | 固定 15% 宽度 (scale=iw*0.15) | ✅ |
| 生成数量 | int | 否 | 独立页面 1-4 张；团队任务每篇文章生成 1 张 | 待前端 |
| 跳过Logo | bool | 否 | skipLogo 参数可跳过硬水印图片 | ✅ |

### 4.2 输出

- **格式**：PNG / JPG
- **分辨率**：不低于 1080px 短边，建议 2K
- **水印**：Logo 以半透明 PNG 叠加，支持指定位置
- **交付**：单张下载 / 批量打包下载

### 4.3 海报尺寸建议

| 场景 | 尺寸 | 比例 |
|------|------|------|
| 小红书 / 抖音竖版 | 1080×1440 | 3:4 |
| 微信公众号封面 | 900×383 | 2.35:1 |
| 微信朋友圈 | 1080×1080 | 1:1 |
| 活动横幅 | 1920×720 | 8:3 |
| 通用竖版 | 1080×1920 | 9:16 |

### 4.4 Logo 水印逻辑

- 读入品牌 Logo 文件（从 `video_logo_url` 设置项，与视频共用）
- 等比缩放到宽度 15%（`scale=iw*0.15`）
- 叠加位置：右上角，距右 20px 距顶 20px（`overlay=W-w-20:20`）
- 实现方式：ffmpeg image overlay（与视频水印共用同一套逻辑）
- 水印后原图保留 `_original` 备份，覆盖生成带水印版本
- 实现文件：`server/poster-api.cjs` → `applyLogoOverlay()`

### 4.5 集成场景

**场景 A：团队任务自动生成（已实现）**
- 创建团队任务时选择「📝 文案+图片」类型 → 勾选「🎨 生成海报」
- 任务执行时 copywriter 完成后，imagegen 阶段对每篇文章调用 `genPoster()`
- 使用文章的 `imagePrompt`（AI 生成的视觉描述）作为海报 prompt
- 海报尺寸/风格默认，可通过 `task.config.posterSize` / `task.config.posterStyle` 扩展

**场景 B：独立海报生成页面（规划中）**
- 侧边栏「海报生成」入口，与 TTS、视频并列
- 支持手动输入文案 + 选风格/尺寸/数量
- 结果预览 + 下载 + 历史记录

### 5.3 关键交互

1. **一键生成**：填写文案后直接点击生成，使用默认配置
2. **高级配置**：折叠面板，不影响核心流程
3. **生成中状态**：骨架屏 + 进度提示
4. **结果预览**：点击海报可放大查看
5. **不满意重试**：可选择「保留文案，换一批风格」或「编辑文案，重新生成」
6. **历史记录**：按时间倒序展示，支持下载和删除

## 6. 技术方案概要

### 6.1 后端
- **API 路由**：`POST /api/ai/poster`（新）；`POST /api/ai/image` 兼容转发
- **AI 服务**：`server/poster-api.cjs` — Seedream 图生图 + Logo 叠加
- **API 调用**：Volcengine Ark `POST api/v3/contents/generations/tasks`（与 Seedance 统一入口）
- **水印处理**：ffmpeg image overlay（复用视频水印逻辑）
- **存储**：下载到本地 DATA_DIR，通过 `/api/file/` 提供访问
- **团队集成**：`team.cjs` imagegen 阶段改 skip → 可选海报生成
- **WebSocket 推送**：ai.cjs 中原有 image 任务的 WS 进度推送全部保留

### 6.2 前端
- 输入表单：React + Tailwind CSS
- 图片预览：lightGallery / 自定义 Modal
- 状态管理：React Query（请求缓存和轮询）
- 历史记录：分页加载

### 6.3 数据模型

```sql
CREATE TABLE posters (
  id            UUID PRIMARY KEY,
  user_id       VARCHAR(64),
  prompt        TEXT,           -- 用户输入文案
  config        JSONB,          -- 风格/尺寸/配色等配置
  images        JSONB,          -- [{url, size, ...}]
  status        VARCHAR(32),    -- pending/processing/done/failed
  created_at    TIMESTAMP
);
```

## 7. 验收标准

| 编号 | 验收项 | 标准 |
|------|--------|------|
| AC-1 | 文案输入 | 支持输入 1-200 字文案，超出提示限制 |
| AC-2 | AI 生成 | 输入文案后 30s 内生成至少 1 张可用的海报 |
| AC-3 | Logo 水印 | 生成海报须包含品牌 Logo，位置/大小与配置一致 |
| AC-4 | 多尺寸 | 至少支持 3 种主流尺寸（小红书、朋友圈、横版封面） |
| AC-5 | 多风格 | 至少支持 3 种风格切换（简约/促销/科技） |
| AC-6 | 批量下载 | 生成多张时支持批量打包下载 |
| AC-7 | 历史记录 | 用户可在历史记录查看、下载、删除过往海报 |
| AC-8 | 错误处理 | AI 生成失败时有友好提示和重试入口 |
| AC-9 | 图片质量 | 生成图片分辨率≥1080px，无明显变形/模糊/文字错乱 |
| AC-10 | 响应式 | 页面在桌面端良好显示，移动端可查看历史记录 |
