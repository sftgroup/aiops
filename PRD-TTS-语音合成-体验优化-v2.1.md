## 1. 优化背景

v1.0 TTS 功能已上线，核心流程（文案→翻译→语音合成→下载）可正常工作。经用户体验反馈，识别出以下体验问题：

| # | 问题 | 影响 |
|---|------|------|
| 1 | 切换音色后，之前的播放/下载结果丢失 | 用户来回对比音色时需要反复生成，体验割裂 |
| 2 | 历史文案面板为空 | 右侧面板形同虚设，无法实现「一键填入历史文案」 |
| 3 | 翻译文案缺乏独立展示和下载 | 翻译结果混在结果卡片中，用户无法单独使用译文 |
| 4 | 缺乏 AI 辅助能力 | 用户需自行调整文案节奏和选择音色，缺乏智能引导 |
| 5 | 优化后文案无处展示 | 用户看不到 AI 优化前后的对比 |

## 3. 详细需求

### REQ-01 — 切换音色/语言不丢失结果

**现状问题**：
用户生成语音后，切换语言或音色按钮会调用 `setResult(null)`，导致当前播放区域和下载按钮立即消失。用户想对比不同音色的效果时，每次切换语言都要重新生成，操作繁琐。

**期望行为**：

```
用户操作流程：
  1. 输入文案 → 选择「中文·晓晓」→ 生成语音 → 看到结果卡片（可播放/下载）
  2. 切换语言到「English」→ 语音列表更新，但结果卡片保留不消失
  3. 切换音色到「Jenny」→ 结果卡片同样保留
  4. 再次点击「生成语音」→ 用新参数生成，结果卡片替换为新结果
```

**实现要点**：
- 切换语言/音色时**不再** `setResult(null)`
- 切换语言时自动选择该语言第一个可用语音（已有逻辑，不需要改）
- 结果卡片始终显示最后一次成功生成的结果，直到新的生成覆盖
- 如果用户手动清空/修改文案，结果卡片保留（因为可能只是想调整文案，旧的音频结果仍有参考价值）

### REQ-02 — 历史文案面板加载 TTS 历史

**现状问题**：
右侧「历史文案」面板读取的是 `contentStore.contents`（来自文案生成模块的全局内容列表），但在 TTS 页面中这个列表可能是空的（用户不一定先去了文案生成页面），导致面板显示「暂无历史文案」。

**根因分析**：
- `contents` 来自 `ContentPage` 的全局内容加载，并非 TTS 专属历史
- TTS 生成记录虽然写入了 `contents` 表（`type='tts'`），但 `ContentPage` 的列表加载可能只筛选了非 tts 类型，所以 TTS 历史记录不在前端 store 中

**期望行为**：
右侧面板改名为「TTS 历史」，展示用户**最近 20 条 TTS 生成记录**（按时间倒序），包含：
- 原文预览（前 60 字）
- 目标语言
- 生成时间
- 操作按钮：「填入文案」「播放」

**后端新增 API**：

```
GET /api/tts/history?limit=20
Authorization: Bearer <token>

Response:
{
  "items": [
    {
      "id": "uuid",
      "text": "原文内容...",
      "translatedText": "翻译内容...",
      "lang": "en-US",
      "voice": "en-US-JennyNeural",
      "url": "/api/file/tts_uuid.mp3",
      "createdAt": 1719300000000
    }
  ]
}
```

**实现要点**：
- 从 `contents` 表中查询 `type='tts'` 且 `userId` 匹配的记录
- 按 `createdAt` 倒序，默认返回 20 条
- 前端点击「填入文案」→ 填入原文到输入框（不清除当前结果）
- 前端点击「播放」→ 直接播放该历史记录对应的 MP3

### REQ-03 — 翻译结果独立展示框 + 下载译文

**现状问题**：
翻译后的文本只在「生成语音」的结果卡片中显示，且只有复制按钮，无法单独下载译文。如果用户只需要翻译不需要合成语音，也没有单独的入口。

**期望行为**：

新增**独立翻译功能区**，包含以下交互：

**3.1 独立翻译按钮**

在「生成语音」按钮旁边新增「仅翻译」按钮：
```
[🎤 生成语音]  [🌐 仅翻译]
```

**3.2 翻译结果展示框**

点击「仅翻译」后，在输入框下方出现翻译结果卡片：
```
┌─────────────────────────────────────────────────────┐
│  🌐 翻译结果 （en-US · English）                        │
│  ┌─────────────────────────────────────────────────┐ │
│  │  This is the translated text...                  │ │
│  └─────────────────────────────────────────────────┘ │
│  [📋 复制]  [⬇️ 下载 TXT]  [🎤 用此译文合成语音]        │
└─────────────────────────────────────────────────────┘
```

**3.3 下载译文 TXT**

点击「下载 TXT」→ 浏览器下载 `translation_{timestamp}.txt`，内容为纯翻译文本。

**后端利用**：
- 复用已有 `POST /api/tts/translate` 端点（已存在，无需新增）
- 新增 `GET /api/tts/download-text/:id` — 按记录 ID 下载译文 TXT

**实现要点**：
- 翻译结果与合成结果是独立状态（`translationResult` ≠ `ttsResult`）
- 用户可以先翻译再合成，也可以直接合成（翻译在合成流程中自动完成）
- 从翻译结果点击「用此译文合成语音」→ 带译文发起 TTS 合成（跳过翻译步骤）

### REQ-04 — AI 智能优化文案（长句→短句）

**现状问题**：
用户输入的长文案可能包含长句、复杂句式，合成语音听起来不够自然。AI 可以帮助优化文案节奏。

**期望行为**：

在输入框上方新增「AI 优化」按钮：

```
[✨ AI 优化文案]
```

点击后调用 DeepSeek，对当前文案进行口语化优化：

**优化要求**（system prompt）：
```
你是专业的配音文案优化师。对文案做以下优化：
1. 将长句拆分为短句（每句不超过20字）
2. 维持原文的核心信息和语气
3. 移除不适合朗读的标点/符号
4. 使用口语化表达，适合自然朗读
5. 只返回优化后的文案，不要解释
```

**优化结果展示**：

优化完成后，在输入框下方出现对比卡片：
```
┌─────────────────────────────────────────────────────┐
│  ✨ AI 优化结果                                        │
│                                                       │
│  【优化前】                                            │
│  欢迎关注我们的频道，今天我们将探索人工智能的无限可能...    │
│                                                       │
│  【优化后】                                            │
│  欢迎关注我们的频道。今天，我们来探索人工智能。            │
│  它的可能性，是无限的。                                  │
│                                                       │
│  [📋 复制优化版]  [⬆️ 替换输入框]  [🎤 用优化版合成语音]  │
└─────────────────────────────────────────────────────┘
```

**后端新增 API**：

```
POST /api/tts/optimize
Authorization: Bearer <token>
Body: { "text": "原文" }

Response:
{
  "original": "原文",
  "optimized": "优化后文案"
}
```

**实现要点**：
- 优化过程显示 loading 态（按钮显示 `⏳ 优化中...`）
- 优化结果不允许覆盖（多次优化产生多个结果，用户可对比选择）
- 优化是一个纯文本处理，不涉及 TTS

### REQ-05 — AI 智能推荐音色

**现状问题**：
用户面对 20+ 种语言和每种语言下的 2-15 个音色，选择困难。AI 可以根据文案内容自动推荐合适的音色。

**期望行为**：

在音色选择区新增「🤖 AI 推荐音色」按钮：

点击后，DeepSeek 分析文案内容的情感倾向和用途，推荐最匹配的音色：

**推荐逻辑**（system prompt）：
```
你是配音导演。根据文案的内容和风格，从以下音色中推荐最适合的3个：
{可供选择的音色列表}

请分析：
1. 文案情感基调（例如：新闻播报、产品推广、故事讲述、教育培训等）
2. 推荐理由（简短）

返回 JSON 格式：
{
  "tone_analysis": "产品推广，热情有力",
  "recommendations": [
    {"voiceId": "en-US-JennyNeural", "reason": "女声亲切有力，适合促销文案"},
    {"voiceId": "en-US-GuyNeural", "reason": "男声稳重可信，适合品牌推广"},
    {"voiceId": "en-US-AriaNeural", "reason": "女声清晰明亮，适合产品介绍"}
  ]
}
```

**推荐结果展示**：

在音色选择区下方出现推荐卡片：
```
┌─────────────────────────────────────────────────────┐
│  🤖 AI 推荐音色   基调分析：产品推广，热情有力            │
│                                                       │
│  ⭐ 推荐 1：Jenny（女·美式）                             │
│  女声亲切有力，适合促销文案             [选择此音色]     │
│                                                       │
│  ⭐ 推荐 2：Guy（男·美式）                               │
│  男声稳重可信，适合品牌推广             [选择此音色]     │
│                                                       │
│  ⭐ 推荐 3：Aria（女·美式）                              │
│  女声清晰明亮，适合产品介绍             [选择此音色]     │
└─────────────────────────────────────────────────────┘
```

**后端新增 API**：

```
POST /api/tts/recommend-voice
Authorization: Bearer <token>
Body: { "text": "文案内容", "targetLang": "en-US" }

Response:
{
  "tone_analysis": "...",
  "recommendations": [
    { "voiceId": "en-US-JennyNeural", "name": "Jenny (女·美式)", "reason": "..." }
  ]
}
```

**实现要点**：
- AI 推荐前需要用户选择目标语言（因为不同语言音色不同）
- 推荐结果可点击直接设置音色
- 如果文案为空，按钮置灰不可点击
- 同语言的音色列表从服务端动态获取后传给 AI

### REQ-06 — 优化后文案独立展示框

**现状问题**：
对应 REQ-04。AI 优化后的文案需要独立展示框，用户可以对比查看、复制、替换。

**期望行为**：

见 REQ-04 中的「优化结果展示」卡片设计。

**与 REQ-04 的关系**：
- REQ-04 定义 AI 优化能力和后端 API
- REQ-06 定义优化结果的前端展示 UI
- 两者在开发时合并为一个任务

**额外交互**：
- 用户可以多次点击「AI 优化」生成多个优化版本（每次替换优化结果卡片）
- 如果用户修改了原文后再次优化，之前的结果被替换

## 4. 技术方案

### 4.1 后端新增/修改

| API 端点 | 方法 | 说明 | 新增/修改 |
|----------|------|------|----------|
| `/api/tts/history` | GET | 获取用户 TTS 历史记录 | **新增** |
| `/api/tts/optimize` | POST | AI 优化文案（口语化） | **新增** |
| `/api/tts/recommend-voice` | POST | AI 推荐音色 | **新增** |
| `/api/tts/download-text/:id` | GET | 下载译文 TXT 文件 | **新增** |
| `/api/tts/generate` | POST | 支持 `skipTranslation` 参数 | **修改** |

### 4.2 前端修改

| 文件 | 变更 |
|------|------|
| `TtsPage.tsx` | 核心改动：新增翻译/优化/推荐状态和 UI |

### 4.3 数据流（新增）

```
┌─ AI 优化 ──────────────────────────────┐
│ 用户点击「✨ AI 优化」                    │
│  → POST /api/tts/optimize { text }     │
│  → DeepSeek 口语化优化                   │
│  → 返回 { original, optimized }        │
│  → 前端展示优化对比卡片                   │
└─────────────────────────────────────────┘

┌─ AI 推荐音色 ───────────────────────────┐
│ 用户点击「🤖 AI 推荐音色」                 │
│  → POST /api/tts/recommend-voice       │
│    { text, targetLang }                │
│  → DeepSeek 分析文案 + 推荐音色          │
│  → 返回 { tone_analysis,               │
│     recommendations[] }                │
│  → 前端展示推荐卡片，可点击选择            │
└─────────────────────────────────────────┘

┌─ 历史文案加载 ──────────────────────────┐
│ TtsPage 挂载时（或切换 tab 时）           │
│  → GET /api/tts/history?limit=20       │
│  → 返回 TTS 专属历史记录                  │
│  → 渲染右侧历史面板                       │
└─────────────────────────────────────────┘
```
